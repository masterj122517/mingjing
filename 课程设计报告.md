# 明镜 · 智能学习计划管理系统

> 课程设计报告

---

## 一、系统概述

### 1.1 项目背景

在碎片化学习时代，学习者面临以下痛点：

- **目标模糊**：只有笼统的想法（"我要学英语"），缺乏结构化拆解
- **执行失焦**：每天不知道具体该做什么，容易拖延
- **进度盲区**：不知道自己学得怎么样，哪些地方需要调整

明镜（Mingjing）正为解决这些问题而设计——它帮助用户设定学习目标，通过 AI 自动拆解为可执行的任务序列，并持续追踪进度、提供智能反馈。

### 1.2 系统定位

- **产品形态**：桌面端应用（macOS / Windows / Linux）
- **目标用户**：需要结构化学习计划的自学者、备考学生
- **核心价值**：从"设定目标"到"完成任务"的全流程智能管理

### 1.3 核心功能

| 功能模块 | 说明 |
|---------|------|
| 目标管理 | 创建学习目标，设截止日期，跟踪完成状态 |
| 任务管理 | 标准 CRUD，支持优先级、标签、分类、截止日期 |
| AI 规划 | 基于 LLM 自动将目标拆解为 5-15 个结构化任务 |
| AI 分析 | 智能诊断进度风险，输出建议和进度总结 |
| 统计图表 | 完成率、每日趋势、优先级分布等可视化 |
| PDF 导出 | 一键导出学习报告 |
| 离线规则 | 无 AI 时内置规则引擎兜底 |
| 数据加密 | AES-256-GCM 加密存储 API Key |

## 三、系统设计

### 3.1 技术选型

| 层次 | 技术 | 选型理由 |
|------|------|---------|
| 桌面框架 | Tauri 2 | 相较 Electron，体积小(~5MB vs ~150MB)、性能优、安全性高 |
| 后端语言 | Rust | 内存安全、零成本抽象、与 Tauri 原生集成 |
| 数据库 | SQLite (rusqlite) | 零配置、嵌入式、适合桌面单机场景 |
| 前端 | Vanilla HTML/CSS/JS | 零构建步骤、无框架锁定、与 Tauri 配合轻量 |
| HTTP 客户端 | reqwest | Rust 生态最流行的异步 HTTP 库 |
| 加密 | ring (AES-256-GCM) | Rust 原生、经过安全审计的密码学库 |
| 图表 | Chart.js | 轻量(~70KB)、功能完备的 JavaScript 图表库 |
| PDF | jsPDF + html2canvas | 纯前端 PDF 生成，无需后端服务 |

**为什么选 Tauri 而不是 Electron？**

| 对比维度 | Tauri 2 | Electron |
|---------|---------|----------|
| 安装包体积 | ~5 MB | ~150+ MB |
| 内存占用 | ~50 MB | ~200+ MB |
| 安全性 | 沙箱化，Rust 后端 | Node.js 全权限 |
| 开发语言 | Rust + 任意前端 | Node.js + 任意前端 |
| 启动速度 | 快 | 较慢 |

### 3.2 系统架构

```
┌─────────────────────────────────────────┐
│          Frontend (Vanilla JS)           │
│  index.html / main.js / styles.css       │
│  Chart.js / jsPDF / html2canvas          │
│         │ invoke()                       │
│         ▼                                │
├─────────────────────────────────────────┤
│      Tauri 2 Bridge (IPC)                │
│  commands.rs ←→ lib.rs (注册)            │
├──────────┬───────────────┬──────────────│
│          │               │               │
│  db.rs   │  llm.rs      │  rules.rs     │
│  SQLite  │  reqwest     │  离线规则引擎   │
│  CRUD    │  JSON Mode   │  逾期检测      │
│          │               │               │
│  passkey.rs (AES-256-GCM 加密)           │
└──────────┴───────────────┴───────────────┘
```

**数据流（以"AI 规划"为例）：**

```
用户点击目标 → 输入描述
    ↓ invoke('generate_plan', { goal_id, instruction })
Rust 后端收到
    ↓ 查数据库(目标信息) → 查模型配置
    ↓ reqwest 调用 LLM API (JSON Mode)
LLM 返回结构化 JSON
    ↓ 解析为 AnalysisOutput { suggestions: [...] }
    ↓ 返回前端
前端渲染可编辑任务卡片 → 用户审核 → 采纳 → 批量创建
```

### 3.3 数据库设计

**ER 关系：**

```
goals ──┐              categories ──┐
        │                          │
        │ 1:N                      │ 1:N
        │                          │
        └── todos ◄──┘
              │
              └── suggestion (通过 payload JSON)
```

**表结构：**

```sql
-- goals: 学习目标
CREATE TABLE goals (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  title       TEXT NOT NULL,
  description TEXT DEFAULT '',
  created_at  TEXT NOT NULL,
  target_date TEXT,
  status      TEXT NOT NULL DEFAULT 'active'
);

-- categories: 任务分类
CREATE TABLE categories (
  id   INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  icon TEXT NOT NULL DEFAULT 'target'
);

-- todos: 任务明细
CREATE TABLE todos (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  title         TEXT NOT NULL,
  priority      TEXT NOT NULL DEFAULT 'medium',
  tags          TEXT DEFAULT '',
  category_id   INTEGER DEFAULT NULL,
  goal_id       INTEGER DEFAULT NULL,
  created_at    TEXT NOT NULL,
  due_at        TEXT,
  completed_at  TEXT,
  FOREIGN KEY (category_id) REFERENCES categories(id),
  FOREIGN KEY (goal_id) REFERENCES goals(id)
);

-- model_configs: LLM 模型配置
CREATE TABLE model_configs (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  name       TEXT NOT NULL,
  provider   TEXT NOT NULL,
  api_base   TEXT NOT NULL,
  api_key    TEXT NOT NULL,
  model_name TEXT NOT NULL,
  is_active  INTEGER NOT NULL DEFAULT 0
);

-- suggestions: AI 建议记录
CREATE TABLE suggestions (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  suggestion_type TEXT NOT NULL,
  todo_id         INTEGER,
  payload         TEXT NOT NULL,
  batch_id        TEXT NOT NULL,
  status          TEXT NOT NULL DEFAULT 'pending',
  created_at      TEXT NOT NULL
);
```

## 四、关键实现

### 4.1 AI 学习计划生成

核心在 `llm.rs`，采用 OpenAI 兼容 API 的 JSON Mode：

```rust
// 构造 prompt → 要求 LLM 只返回 JSON
const PLAN_PROMPT: &str = r#"你是"明镜"学习计划管理系统的 AI 导师...
// 输出格式必须是严格的 JSON，包含 analysis 和 suggestions
// LLM 返回 JSON → 直接反序列化为 AnalysisOutput
"#;
```

关键设计决策：
- LLM API Key 可加密存储（AES-256-GCM），也可明文
- JSON Mode 确保 LLM 返回结构化数据，直接映射到 Rust struct
- LLM 调用失败自动回退到 `rules.rs` 规则引擎

### 4.2 离线规则引擎

`rules.rs` 实现了无需 AI 的基础分析：

```rust
pub fn offline_analysis(todos, goals) -> AnalysisOutput {
    // 扫描所有任务，标记逾期、临近截止
    // 统计总量、完成量
    // 返回风险预警 + 进度总结
}
```

保证无网络环境下核心功能仍然可用。

### 4.3 API Key 加密

`passkey.rs` 使用 `ring` 库的 AES-256-GCM 对 API Key 加密：

```
用户输入密码 → SHA-256 派生 256 位密钥
    → AES-256-GCM 加密 API Key
    → base64 编码存入库中
验证密码 → 比对 SHA-256 哈希
重加密 → 解密再加密所有密钥
```

### 4.4 数据迁移

首次启动时自动迁移旧的 localStorage 数据：

```
loadData() → 检测 DB 是否为空
    → 是 → 调用 migrate 命令
        → 读取前端 localStorage（如有）
        → 解析为 MigratePayload
        → 写入 SQLite
```

### 4.5 统计与 PDF

```
get_statistics (Rust):
    SELECT COUNT(*), SUM(CASE WHEN ...) 等聚合 SQL
    → 返回 Statistics JSON

前端接收 → Chart.js 渲染柱状图 + 折线图
    → html2canvas 截图图表
    → jsPDF 生成 A4 PDF 报告
```

## 六、总结

### 6.1 项目亮点

1. **AI 驱动的计划生成** — 利用大语言模型将模糊目标自动拆解为结构化任务，显著降低规划成本
2. **双层智能分析** — LLM 在线分析 + 规则引擎离线兜底，保证可靠性
3. **安全优先** — API Key 使用 AES-256-GCM 加密，主密码保护
4. **现代桌面体验** — Tauri 2 提供原生级性能，毛玻璃暗色主题提升视觉品质
5. **零构建前端** — 纯净 HTML/CSS/JS 架构，降低开发和维护复杂度

### 6.2 可改进方向

- 多用户支持与数据同步
- 日历视图与拖拽排期
- 学习提醒（系统通知）
- 更丰富的统计图表（饼图、热力图）
- 导入/导出标准格式（CSV / iCal）
